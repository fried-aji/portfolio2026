# CLAUDE.md

## 使用する言語

必ず日本語を使用してください。文字化けに注意し、文字コードはUTF-8を使用してください。

## 命令を受けてからコードを編集するまでの流れ

1.  ユーザーからの命令を受け取る
2.  受け取った命令を理解し、何を達成すべきかユーザーに確認する
3.  達成すべき目的のために作成・編集・削除するファイルを、根拠を添えて列挙し、ユーザーに確認する
4.  ユーザーからの確認を受け取ったら、命令の目的を達成するための詳細な実行計画をユーザーに提示する
5.  ユーザーからの確認を受け取ったら、実行計画に基づいてコードを編集する

## コーディングルール

### パッケージ管理

バージョン管理ツールにmise、パッケージマネージャーはpnpmを使用しています。

### Lintコマンド

コード編集後は以下のコマンドを使用してLintエラーが出ていないかチェックしてください。実行するコマンドは編集したコードに合わせて使い分けてください。

| `pnpm run lint:html` | HTML チェック（Markuplint） |
| `pnpm run lint:es` | JavaScript チェック（ESlint） |
| `pnpm run lint:style` | CSS チェック（Stylelint） |
| `pnpm run lint:check` | HTML・CSS・JS チェック |

### ディレクトリ構成

README.mdを参照してください。

## 実行時の厳守事項

- **影範囲の確認**：意図しない変更や影響が発生する箇所を特定して作業を実施してください。
- **人昧さの回避**：「一の可能性があります」「〜かもしれません」を禁句とし、必ず一次情報を根拠に作業してください。根拠となるソースのURLと該当箇所の引用を必ずユーザーに知らせてください。
- **指示にないスタイリングの追加**：独断でUIのデザインを変更、追加しないでください。
- **エラーの際献**：失敗しているテストをskipしたり、削除したり、エラーを隠蔽したりするような行為は禁止とします。
